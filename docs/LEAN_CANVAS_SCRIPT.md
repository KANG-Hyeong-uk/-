# Trust 린캔버스 발표 대본 (6분)

> KAIST E5 예선 발표용
> 린캔버스 v3 기준

---

안녕하십니까. 스톤즈랩 정진협입니다.
지금부터 Trust 린 캔버스 발표 시작하겠습니다.

단 5분, 단 $10로 점검하는 바이브코더를 위한, 바이브코더에 의한, 바이브코더의 보안스캐너.
Trust입니다.

보안서비스 Trust는 클로드, GPT 같은 AI로는 찾을 수 없는 취약점을,
URL 입력 한 번이면 5분 내로 점검합니다.

저희 팀은 Trust로
비싸고 어려운 보안과 모의해킹을 이제 바꿔나가겠습니다.


## 1. 문제 (0:00–0:40)
한 가지 질문으로 시작해보겠습니다.
혹시 온라인에 서비스 배포하기 전에 보안 점검, 하고 계시나요?

2025년 보안 전문기업 조사에 따르면, 개발자의 86%가 보안을 후순위로 둡니다.
시간도 없고, 방법도 모르고, 도구는 비싼데 티도 안납니다.

대부분 AI한테 그냥 "우리 서비스 보안은 어때?" 정도로 물어보고 끝납니다.
AI가 코드를 읽고 "여기 위험해 보여요"라고 말해주면 패턴대로 수정하는 것 외에는 없습니다.

그런데 진짜 문제는, AI는 코드를 읽을 수만 있다는 겁니다.

실제로 내 사이트에 접속해서 보안 인증서가 만료됐는지, 서버에 헤더가 노출되는지, API에 SQL Injection이 가능한지.. 이건 코드를 아무리 잘 읽어도 알 수 없습니다.

배포된 서비스의 취약점은 코드가 아니라, 실제로 접속해봐야 알 수 있습니다.

---

## 2. 고객군 (0:40–1:10)
이 문제를 가장 크게 겪는 사람들이 바로 저희 바이브 코더입니다.

전 세계에 약 1,400만 명이 Cursor, Claude Code 같은 AI로 코딩하고 직접 배포하고 있습니다.

이들의 특징은 명확합니다.
만드는 건 빠른데, 보안을 고려하거나 의뢰할 예산은 없습니다.
2026년 기준으로 모의해킹은 중저가로 알아봐도 한 번에 500만 원입니다. 
보안으로 유명한 소나큐브나 스닉의 구독료는 최소 월 $125입니다.
비용을 보시면 알겠지만, 1인 개발자가 접근할 수 있는 보안 도구가 사실상 없습니다.
그런데 개인정보가 탈취당하면? 벌금으로 이정도는 훌쩍 넘는 금액을 지불해야 합니다.

저희는 여기서 기회를 발견했습니다.

---

## 3. 핵심 가치 제안 + UVP (1:10–2:00)

그래서 저희는 만들었습니다.
보안서비스 Trust입니다.

Trust가 다른 보안 도구와 근본적으로 다른 점은 4가지입니다.

첫째, 내 사이트에 해커가 들어올 수 있는지 실제로 확인해줍니다.
코드를 읽는 게 아니라, 진짜 접속해서 테스트합니다.

둘째, 내가 안 해도 알아서 매일 점검하고 알려줍니다.
"보안 해야지"라고 결심할 필요가 없습니다. 이미 돌아가고 있습니다.

셋째, 문제를 찾으면 고치는 코드까지 만들어줍니다.
"여기 위험합니다"에서 끝나는 게 아니라, "이렇게 고치세요"까지 갑니다.

넷째, 모의해킹 한 번에 500만 원인데, Trust는 월 $9.90입니다.

---

## 4. 솔루션 (2:00–2:50)

이 Trust가 문제를 해결하는 방법은 다음과 같습니다.

URL을 입력하면, 5,000개 이상의 공격 템플릿으로 HTTP 요청을 보냅니다.
SQL Injection, 보안인증서 설정 오류 등등 전부 테스트합니다.

취약점이 발견되면 AI가 수정 코드를 생성하고,
버튼 한 번이면 GitHub에 수정 PR이 올라갑니다.

여기서 끝이 아닙니다.

자동 정기 스캔을 설정하면 매일, 매주 알아서 돌아가고,
문제가 발견되면 이메일로 바로 알려줍니다.

GitHub을 연동하면 PR을 올릴 때마다 자동으로 보안 검수가 됩니다.

이 핵심은, 완전히 심리스하게, 고객은 행동을 바꾸지 않아도 된다는 것입니다.

---

## 5. 경쟁우위 (2:50–3:20)

올해 보안시장의 가장 큰 이슈는
클로드 코드 시큐리티의 출시일 것입니다.

올해 2월 20일 이후 "Claude Code Security가 나왔는데 Trust가 필요한가?"라는 질문을 많이 받았습니다.

근데 답은 간단합니다. 둘은 하는 일이 다릅니다.

Claude Code Security는 코드를 읽고 추론하는데요,
반면에 Trust는 배포된 서비스에 실제로 접속해서 뚫리는지 테스트합니다.

트러스트가 온라인에서 보는 XSS나 보안인증은
코드를 아무리 잘 읽어도 알 수 없는 영역입니다.

Claude가 코드를 검토하고, Trust가 배포된 서비스를 검증한다.
경쟁이 아니라 보완 관계입니다.

그리고 가격입니다.
1만원대 가격으로 매일 점검이 가능합니다.

## 6. 시장 (3:20–3:50)

저희가 집중하는 시장은 대한민국이 아닌, 전세계입니다.
글로벌 보안 시장은 2025년 기준 337억 달러, 연 18%씩 성장하고 있습니다.

이 중 보안 도구 시장이 약 50억 달러.
AI 코딩 사용자 4,700만 명의 1%만 유료 전환해도 5600만 달러 규모입니다.

AI가 코드 분석을 맡기 시작한 지금, 배포 후 검증 수요가 폭발할 것으로 전망하고 있습니다.

## 7. 수익원 + 유닛 이코노믹스 (3:50–4:40)

비즈니스 모델은 구독형 SaaS입니다.

B2C Pro Plan이 월 $9.9,
B2B Team Plan은 시트당 월 $29로 준비 중입니다.
글로벌 100개국 이상에서 결제가 가능합니다.

트러스트의 유닛 이코노믹스로 넘어가겠습니다.

스캔 1회 원가는 $0.065입니다.
서버 비용과 API 비용을 합산한 금액입니다.

Pro 유저의 월 순매출은 $8.90. 구독료에서 결제 수수료를 뺀 금액입니다.

유저가 월 10회 스캔하면 마진 93%, 30회 스캔해도 78%입니다.
적자가 나려면 한 사람이 월 137회를 스캔해야 합니다.

현재 시뮬레이션으로 Pro 유저 12명이면 인프라 고정비를 커버합니다.

## 8. 채널 + 비용구조 (4:40–5:30)

시장 진출 후 고객 획득 전략은 다음과 같습니다.

Product Hunt 글로벌 런칭으로 초기 유입을 만들고,
MCP 마켓플레이스와 Reddit, Hacker News에서 개발자 커뮤니티에 먼저 노출합니다.

그리고 저희는 첫 경험의 마찰을 제거했습니다.
로그인 없이도 URL만 넣으면 바로 결과를 확인할 수 있는 경험을 제공합니다.

무료 8회 사용 후 가치를 느끼면 Pro로 전환할 수 있게했고,
GitHub에 보안 뱃지를 달거나 리포트를 공유하면서 자연스럽게 바이럴이 됩니다.

비용 구조입니다.

현재 월 고정비는 $100도 채 안됩니다.
여기에 API 비용이 사용량에 따라 $15에서 $80 추가됩니다.
현재는 Pro 유저 1명의 공헌이익으로 Free 유저 25명을 커버할 수 있습니다.

## 9. 마무리 (5:30–6:00)

정리하겠습니다.

개발자의 86%가 보안을 하지 않습니다.
하지 않는 게 아니라, 할 수 없었던 겁니다.
비싸고, 어렵고, 귀찮습니다.

Trust는 이 세 가지를 동시에 해결합니다.

이제는 AI가 코드를 작성하고, 검토하고, 배포하는 시대가 왔습니다.
다음은 배포된 서비스를 검증하는 시대입니다.

Trust, 그리고 팀 스톤즈랩이 그 시작점에 서 있습니다.
감사합니다.
